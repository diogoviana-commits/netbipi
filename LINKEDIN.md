## Short post

Acabei de publicar o NetBIPI, uma plataforma open source para centralizar a operação N1/N2 em um único painel.

A proposta é unir monitoramento, service desk, inventário, diagnóstico de rede, base de conhecimento e relatórios em uma experiência operacional única, com integração com Zabbix, GLPI, WebSocket e Docker.

O objetivo é simples: reduzir troca de ferramentas, acelerar a resposta a incidentes e dar mais contexto para o time.

Stack: React, TypeScript, Node.js, PostgreSQL, Socket.io e Docker.

Se você trabalha com suporte, infraestrutura, observabilidade ou NOC, vale conferir o repositório.

## Long post

Hoje estou publicando o NetBIPI como open source.

Ele foi pensado para times de suporte e infraestrutura que precisam alternar entre Zabbix, GLPI, logs, planilhas e várias telas ao mesmo tempo. Em vez de espalhar a operação por ferramentas diferentes, o projeto concentra a visão em um hub operacional mais claro e mais rápido de consultar.

O NetBIPI reúne:

- Dashboard em tempo real
- Integração com Zabbix e GLPI
- Inventário de ativos e mapa visual
- Diagnóstico de rede direto no navegador
- Base de conhecimento para runbooks
- Relatórios operacionais
- Modo quiosque para sala de operações

Stack principal:
React, TypeScript, Node.js, Express, PostgreSQL, Socket.io e Docker.

O repositório está aberto para quem quiser testar, contribuir ou usar como referência em projetos internos.

## GitHub release text

NetBIPI v1.0.0 marca o primeiro release público da plataforma.

Esta versão entrega uma base open source para centralizar alertas, chamados, ativos, diagnóstico de rede, relatórios e visibilidade operacional em um único lugar. O projeto foi estruturado para ambientes de laboratório e também para servir como ponto de partida em operações reais que precisem de uma interface unificada para N1/N2.
