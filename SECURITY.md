# Security Policy

## Supported versions

The latest public `main` branch is the supported version.

## Reporting a vulnerability

If you find a security issue, do not publish exploit details in a public issue.
Use GitHub Security Advisories for private disclosure whenever possible.

If Security Advisories are not available in your fork or deployment, contact the
project maintainer through a private channel before sharing technical details.

## What to report

- Authentication or authorization bypass
- Secret leakage
- SQL injection or command injection
- Cross-site scripting
- Unsafe defaults in deployment or configuration
- Exposure of tokens, credentials or internal endpoints
